Чтобы отключить запрос подтверждения fingerprint при установлении связи по SSH в Debian, вы можете использовать опции StrictHostKeyChecking и UserKnownHostsFile в файле конфигурации SSH.

1. Откройте терминал и введите следующую команду, чтобы открыть файл конфигурации SSH для редактирования:

sudo nano /etc/ssh/ssh_config

2. В файле ssh_config найдите следующие строки:

#   StrictHostKeyChecking ask
#   UserKnownHostsFile ~/.ssh/known_hosts

3. Удалите символ # в начале строк, чтобы закомментировать их и изменить значение ask и ~/.ssh/known_hosts на no и /dev/null соответственно:

StrictHostKeyChecking no
UserKnownHostsFile /dev/null

4. Сохраните изменения, нажав Ctrl + O, а затем Enter. Затем закройте файл редактора, нажав Ctrl + X.

5. Перезапустите службу SSH для применения изменений, выполните следующую команду:

sudo service ssh restart

Теперь при установлении SSH-соединения в Debian не будет запрашиваться подтверждение fingerprint, и новые fingerprint будут автоматически добавляться в /dev/null, поэтому они не будут записываться или сохраняться.

Обратите внимание, что отключение проверки fingerprint может представлять риск безопасности. Убедитесь, что вы доверяете хосту, с которым устанавливаете SSH-соединение, прежде чем отключать данную проверку.
